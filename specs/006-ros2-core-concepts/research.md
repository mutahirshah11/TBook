# Research Plan: ROS 2 Core Concepts

## Unknowns & Clarifications

1.  **ROS 2 Package Creation Syntax**: Confirm the exact `ros2 pkg create` flags for a Python-only package to ensure best practices.
2.  **Entry Points**: Verify the correct string format for `console_scripts` in `setup.py` to expose nodes as executables.
3.  **Docusaurus Structure**: Confirm how to best structure `docs/part2` in `sidebars.ts` (autogenerated vs manual).

## Research Tasks

- [ ] **Task 1**: Verify `ros2 pkg create --build-type ament_python` syntax.
- [ ] **Task 2**: Check `entry_points` syntax in `setup.py`.
- [ ] **Task 3**: Review Docusaurus sidebar configuration for new subdirectories.

## Findings

### Decision 1: Package Creation
- **Decision**: Use `ros2 pkg create --build-type ament_python <package_name> --dependencies rclpy`
- **Rationale**: This initializes a Python package with `ament_python` build type and adds `rclpy` to `package.xml`.

### Decision 2: Entry Points
- **Decision**: Use `entry_points={'console_scripts': ['executable_name = package_name.module_name:main_function']}`.
- **Rationale**: Standard Python entry point mechanism used by `ament_python`.

### Decision 3: Sidebar Structure
- **Decision**: Add a new category "Part 2: ROS 2 Core Concepts" to `sidebars.ts`, pointing to the generated directory or specific items.
- **Rationale**: keeps navigation clean and grouped.

### Decision 4: Code Examples
- **Decision**: All code examples will be provided as complete, copy-pasteable files within the markdown, but also conceptually linked to a "reference" package structure in the text.
- **Rationale**: Ensures reproducibility.
